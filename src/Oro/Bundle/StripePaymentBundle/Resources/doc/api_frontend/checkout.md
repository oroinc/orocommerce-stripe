# Oro\Bundle\CheckoutBundle\Entity\Checkout

## SUBRESOURCES

### paymentInfoStripePaymentElement

#### add_subresource

Add to the checkout the confirmation token and payment method type required to create a Stripe Payment Intent.

**Important Client-Side Note:**
If you are using the [Stripe.js](https://docs.stripe.com/js) Payment Element on the frontend to generate the confirmation token, you **must** provide the `setupFutureUsage: 'off_session'` option during the confirmation step. This ensures that the payment method can be saved and reused for off-session charges if necessary.

Follow the [Storefront Checkout API Guide](https://doc.oroinc.com/api/checkout-api/#stripe-payment) for more details about the checkout process using the API.

{@request:json_api}
Example:

```JSON
{
  "meta": {
    "confirmationTokenId": "ctoken_1Pxxxxxxxxxxxxxxxxxxxxxx",
    "paymentMethodType": "card"
  }
}
```
{@/request}

### paymentStripePaymentElement

#### add_subresource

Execute checkout payment with Stripe Payment Element.

The Stripe Payment Element method requires the Stripe Confirmation Token and Payment Method Type to be set via the ``paymentInfoStripePaymentElement`` sub-resource before the payment starts.

When the payment is complete, an application should make a second request to the sub-resource to complete the checkout or to properly handle payment errors.

If the payment requires additional actions, such as 3D Secure authentication, the request will fail with a `400 Bad Request` error containing a `requiresAction` flag and additional data required to complete the payment flow on the client side. If the payment is successful, the order resource is returned in the response.

Follow the [Storefront Checkout API Guide](https://doc.oroinc.com/api/checkout-api/#stripe-payment) for more details about the checkout process using the API.

{@request:json_api}
Example of the request:

```JSON
{
  "meta": {
      "successUrl": "https://my-application.ltd/checkout/success",
      "failureUrl": "https://my-application.ltd/checkout/failure",
      "partiallyPaidUrl": "https://my-application.ltd/checkout/partiallyPaid"
  }
}
```

Example of a response when the checkout is not ready for payment:
```JSON
{
    "errors": [
        {
            "status": "400",
            "title": "payment constraint",
            "detail": "The checkout is not ready for payment.",
            "meta": {
                "validatePaymentUrl": "https://oro-application.ltd/api/checkouts/1/payment"
            }
        }
    ]
}
```

Example of a response when payment is still in progress:
```JSON
{
    "errors": [
        {
            "status": "400",
            "title": "payment status constraint",
            "detail": "Payment is being processed. Please follow the payment provider's instructions to complete."
        }
    ]
}
```

Example of a response when payment failed with error:
```JSON
{
    "errors": [
        {
            "status": "400",
            "title": "payment constraint",
            "detail": "Payment failed, please try again or select a different payment method. Error Code: \"card_declined\", Decline Code: \"generic_decline\""
        }
    ]
}
```

Example of a response when additional actions (like 3D Secure) are required:
```JSON
{
    "errors": [
        {
            "status": "400",
            "title": "payment action constraint",
            "detail": "The payment requires additional actions.",
            "meta": {
              "data": {
                "paymentMethod": "stripe_payment_element_7",
                "paymentMethodSupportsValidation": false,
                "errorUrl": "https://oro-application.ltd/payment/callback/error/e111111c-1111-1111-1abc-11dc1d1111f1",
                "returnUrl": "https://oro-application.ltd/payment/callback/return/e111111c-1111-1111-1abc-11dc1d1111f1",
                "failureUrl": "https://my-application.ltd/checkout/failure",
                "partiallyPaidUrl": "https://my-application.ltd/checkout/partiallyPaid",
                "successUrl": "https://my-application.ltd/checkout/success",
                "additionalData": {
                  "confirmationToken": {
                    "id": "ctoken_1Pxxxxxxxxxxxxxxxxxxxxxx",
                    "paymentMethodPreview": {
                      "type": "card"
                    }
                  }
                },
                "successful": false,
                "requiresAction": true,
                "paymentIntentClientSecret": "pi_stripe_secret_key"
              }
            }
        }
    ]
}
```
{@/request}


# Oro\Bundle\StripePaymentBundle\Api\Model\StripePaymentElementPaymentInfoRequest

## FIELDS

### confirmationTokenId

The Stripe Confirmation Token ID generated by the Stripe Payment Element on the client side.

### paymentMethodType

The type of the payment method (e.g., "card", "us_bank_account", "sepa_debit") associated with the confirmation token.


# Oro\Bundle\StripePaymentBundle\Api\Model\StripePaymentElementPaymentRequest

## FIELDS

### successUrl

The URL to which the application should redirect customers after a successful payment completion.

### failureUrl

The URL to which the application should redirect customers when a payment fails.

### partiallyPaidUrl

The URL to which the application should redirect customers when payment is partially paid (if applicable).
